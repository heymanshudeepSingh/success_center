Side Content Bars

This is an example of a side content bar.

The left and right side content bars are meant for supplementary information that is not essential to the page itself.

There is a left and right one. To use them, simply override the Django template block "content_bar_left" or "content_bar_right" to have text in the left and right bars, respectively.

When the bars are empty, there is JavaScript code to remove and hide them. This file can be found in "static/cae_home/js/side_content_bars.js".

Return to Index

Welcome to the CAE Workspace CSS Example Page.

You should only be seeing this page if serving in development mode.


These page contains various, reusable, custom css stylings. Use these for reference when formatting new pages. All custom elements make use of the provided "site themes" that come with the workspace project.

The original intention of these was to have a set of simple CSS styles to use so that all CAE Center projects have a consistent look and feel. While existing CSS frameworks like bootstrap technically accomplish this, we originally felt that bootstrap tends to make html elements "dirty" and hard and was generally overkill for our anticipated needs. Thus, we wanted something that was simpler and less verbose to use.

Of course, like the rest of this "Workspace" project, everything has grown above and beyond initial expectations. We didn't expect this CSS to approach the equivalent of a full, alternative to bootstrap, like it is now. Thus, we now have these pages to showcase and document use of all custom elements that have been created so far.


Generally speaking, the original CSS for these elements can be found at cae_home/static/cae_home/css/sass/content/.

If there is associated JavaScript, then it can be found at cae_home/static/cae_home/js/.

Site Themes

The CSS for this site has various "themes" that the user can select. Selection is available from editing the user profile (when logged in, hover over username in topright corner for nav menu. Then click "user settings"). These themes essentially change the look and feel of the site, by adjusting the colors but not the actual layout.

The original intent was to have "light and dark" versions of the site, but then we needed a "wmu-mockup" look, and the number of themes grew from there.

Site Theme Classes

Most of the elements described in these pages can be customized. In such cases, the elements will have a "default" look, and then additional looks depending specified classes. These classes only change the color of the class, not the layout.

The classes are as follows:

  • Default
  • Primary
  • Secondary
  • Info
  • Success
  • Warning
  • Error
  • Danger

Note that "error" and "danger" are two ways to specify the same stylings.

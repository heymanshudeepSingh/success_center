First Steps

This page lists the general recommended "first steps" in order to start using the CAE Workspace project.

To get started, you should, in order:

  • Read the CAE Workspace Introduction documentation. Specifically, just the points under the "introduction" section. This will give you an idea of what CAE Workspace even is, and why we're using it. It will also give you a starting point to proceed with your own projects.

  • Run the installation script as noted in the helper scripts documentation.

    This script will run all the basic setup steps for your project. Presumably if you're reading this, then you already ran this script. If not, then your system might not be 100% set up to run CAE Workspace, and you might want to go run the script now.

  • Follow the subproject setup documentation, to either create a new subproject, or clone an existing one down. Subprojects are where a majority of useful work is done.

  • Depending on what you want to work on, read the corresponding section of CAE Workspace documentation, which attempts to document all logic that is unique to CAE Workspace.

    If that doesn't seem helpful, also try our GoogleDocs powerpoints, which attempts to show step-by-step, beginner-friendly explanations of full stack development for a standard Django project.

    Finally, feel free to reference the Official DjangoDocs, which are maintained by the Django developers themselves. They cover just about anything you could ever want to know. But be warned that they sometimes can be very verbose, which can be intimidating to those unfamiliar with the Django framework.

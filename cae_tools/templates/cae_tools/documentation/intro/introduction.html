Introduction to the "CAE Workspace"

Preface

The CAE Center is responsible for managing multiple, unrelated workflows and projects for Western Michigan University.

Previously, this was done in the form of multiple websites. Each one had their own HTML, CSS, models, logins, authentication, templating, etc. However, as responsibilities gradually increased, CAE Programmers found themselves spending a large amount of time having to effectively rewrite the same code for more and more individual websites.

Eventually, the CAE Center decided to consolidate everything, so that only one set of "basic templating" needed to be maintained for CSS, models, authentication, etc. Hence, the CAE Workspace was born.

Purpose of the "Workspace" Project

The goal of "CAE Workspace" is to effectively take care of all possible "universal logic", in one single location.

This includes anything that is not unique to any single project maintained by the CAE Center. Such as HTML and CSS, login handling and user authentication, general WMU models such as "Majors" and "Departments", etc.

As result, Workspace itself doesn't exactly...do a whole lot, on its own. But it provides many, many hooks and generalized logic, in hopes of making it simpler to develop various "subprojects". These "subproject" will also be more uniform and consistent in how they handle.

For further information on subproject usage and setup, see General Project Setup.

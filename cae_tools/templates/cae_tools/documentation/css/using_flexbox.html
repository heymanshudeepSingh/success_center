Using FlexBox

Note: When working with flexbox, it's always best to start with the largest, outer-most elements and work inwards.

The "flex-box" style is a (comparatively) newer element display property. It can be given to an element with display: flex; (but see project mixins for a better way to handle this, within Workspace code).

In most situations, the flex-box style, is simply better to used than the original block, inline, or inline-block options. The benefit is that flex-box is a lot more dynamic, and handles screen sizes much more intelligently.

Furthermore, elements (such as divs) with the flex-box style can be nested within each other to create responsive content in as many layers as is needed.

Note that nearly all Workspace CSS effectively defaults to giving elements the flex-box style.

Note on Flex-Grid

There is an alternative style to "flex-box", called "flex-grid". These two function incredibly similar. The main difference is that a flex-box element is designed to only handle one axis at a time (vertical or horizontal). Meanwhile, a flex-grid element is designed to handle both the x and y axis at once.

That sounds superior in theory, right? Why would anyone use a flex-box then? Why not just always use flex-grid?

The drawback to flex-grid is that it's actually much less flexible (no pun intended), overall. Plus, by simply nesting two elements with flex-box, you can get control both vertical and horizontal directions anyway.

To rephrase, flex-grid tends to be more work to maintain, if the element ever needs to change in the future. And it otherwise gives minimal benefit over flex-box. So flex-box is generally just better to use anyway.

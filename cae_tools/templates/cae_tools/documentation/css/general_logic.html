General CSS Logic

Compiling CSS

Since the CAE Workspace CSS is written in Sass, it needs to be compiled on initial project download, and then again every time changes are made. See Custom Scripts for more information.

Sass Files

A majority of CSS for all projects is located in Sass files, found at <project_root>/cae_home/static/cae_home/css/sass/. Within this folder, any files names NOT prefixed with an underscore are considered to be "entrypoint files", and will be read in by the Sass compiler script.

For organization, Sass logic is then divided into separate files, based on the general purpose of the styling. Due to size of the content stylings, these are then further subdivided within their own folder.

Project Sass Variables

Any variables or "universal" logic are found within the variables folder. Anything here is likely used across multiple other Sass files, if not imported into nearly every Sass file found within Workspace.

The _core.scss file merges all other CSS files in the variables folder, as well as contains some functions to make it easier to access various color theme style values (because Sass syntax for accessing these is really gross).

The _colors.scss file contains various color definitions that may or may not be helpful for later reference. Includes things such as "Officially approved WMU colors as of 2018", "colors taken directly from wmich.edu as of 2018", etc.

The _element_groups.scss file contains various definitions of groupings. These are used for looping through elements in other files, to ensure consistency across CSS. Whenever looping through values in Sass, ideally the loop grouping should be defined in this file.

The _font_sizing.scss file contains various sizing definitions for site font options.

The _mixins.scss file contains SASS mixin functions to expand certain CSS values for backwards-browser compatibility. For example, instead of the standard display: flex-box;, one should use @include display-flex; to call this file's backwards-compatible flex-box styles. This is particularly important since the CAE Center serves various sites for WMU, which means we may have to support really old machines that use way outdated browsers.

The _screen_sizing.scss file contains various sizing definitions for site "responsive CSS" stylings (aka, adjusting dynamically based on browser screen size).

    CSS Examples - Nav

    There are 3 distinct nav element types provided by this site:

    • Main Nav
    • Sub Nav
    • Breadcrumb Nav

    Main Nav

    The "main nav" is essentially the site header, aka the bar at the very top of the page.

    On mobile views, this will condense down and have clickable elements, rather than hover elements.

    Generally speaking, this is dictated by the corresponding django project/subproject, and will not change much (or at all) on a page-by-page basis.

    Sub Nav

    The "sub nav" appears as a second bar, directly beneath "main nav" on pages that define it.

    This element should be used for navigation to distinct subpages within a larger "main nav" element. For example, our CAE Web project, has "employee" main nav page. Within this, there are related sub pages such as "employee shifts" and "employee schedules". Both of these sub pages are separate entities from each other, while still belonging within the larger "employee" main nav space. Thus using a sub nav element for this is appropriate.

    To define sub nav on a page, use the sub_nav block within the corresponding page template.

    Breadcrumb Nav

    The "breadcrumb nav" element is meant to show ordering between related paged. It gives the user a means to quickly and easily go back to previous pages.

    This element attempts to display directly underneath the page h1 header tag, if present. If there is no h1 header tag present, then it will instead display directly underneath other page nav elements.

    To define breadcrumb nav on a page, use the breadcrum_nav block within the corresponding page template.
